基本思路：

  1. 用户首次访问提供用户名 + 密码到认证服务器
  2. 服务器验证用户提交信息的合法性，如果验证成功，生成一个token到客户端
  3. 客户端每次访问携带token

token包含信息：

  1. header：

    typ声明类型

    alg生成签名的算法

    { "alg" :"AES256", "typ" :"JWT"}

  2. claims

    sub

    name

    admin

    { "sub":"1234567890", "name":"John Doe", "admin":true}

  3. signature
